## Context

A scalable **analytics platform** built with **Python / FastAPI** and async background jobs that aggregates data from **3+ external APIs** into a unified, multi-tenant pipeline — processing **50K+ daily records** with automated synchronization.

## Problem

Pulling data from several third-party APIs for many tenants is deceptively hard:

- **50+ tenants** need concurrent syncs without overwhelming upstream APIs or each other.
- Schema changes across a multi-tenant database have to apply in the **right dependency order**.
- Users need **real-time visibility** into long-running sync jobs, not a spinner.
- Redundant API calls drive up cost and **rate-limit** pressure.

## Architecture

- **FastAPI + AsyncIO** services running **async background jobs** for synchronization.
- **PostgreSQL** as the multi-tenant store, with **dependency-aware table migrations**.
- **WebSocket** connections for **real-time sync job tracking**, enabling concurrent processing across 50+ tenants.
- **Redis token caching** in front of external API auth to cut redundant calls.

## Implementation

- **Intelligent batching** and **Redis token caching** reduced redundant external API calls.
- **Async background jobs** decouple ingestion from request/response cycles and scale across tenants.
- **WebSocket-based job tracking** gives users a live view of sync progress.
- **Dependency-aware migrations** make multi-tenant schema changes safe and ordered.

## Key Tradeoffs

- **Async workers vs. synchronous requests:** async scales aggregation across tenants but adds coordination and observability complexity.
- **WebSockets vs. polling:** live updates cost a persistent connection, but make long-running jobs feel reliable instead of opaque.
- **Token caching vs. always-fresh auth:** caching slashes redundant calls at the cost of cache-invalidation discipline.

## Results

- Processed **50K+ daily records** from 3+ external APIs.
- Sustained concurrent processing for **50+ tenants** at **99.9% uptime**.
- Reduced redundant API calls by **~60%** via batching and token caching.
- Cut deployment time **~70%** through dependency-aware migrations.

## Lessons

- **Async background jobs are the backbone** of any serious aggregation system.
- **WebSockets convert opaque batch work into a trustworthy, live experience.**
- **Token caching is the cheapest large win** against rate limits and cost.
